    Codebase Map¶

    A developer-oriented reference for navigating the Remitpro source code. Use this as your GPS when you need to find where something lives.

    Frontend: src/¶

    Pages & Routes¶

    All routes are declared in src/App.tsx. The app uses React Router v6.

    URL Path Component File Access Level
    / src/pages/Index.tsx Public (redirects to dashboard if authed)
    /login src/pages/Login.tsx Public
    /onboarding src/pages/Onboarding.tsx Authenticated (new schools only)
    /dashboard src/pages/Admin/Dashboard.tsx Admin+
    /students src/pages/Students.tsx Admin, Bursar
    /students/:id src/pages/StudentDetails.tsx Admin, Bursar
    /finance/* src/pages/Finance/ Admin, Bursar
    /academics/* src/pages/Academics/ Admin, Teacher
    /staff/* src/pages/Staff/ Admin
    /settings src/pages/Settings.tsx Admin
    /reports src/pages/Reports.tsx Admin, Bursar
    /virtual-accounts src/pages/VirtualAccounts.tsx Admin, Bursar
    /superadmin/* src/pages/SuperAdmin/ Super-Admin only
    /portal/:studentId src/pages/Public/ParentPortal.tsx Public (no login)

    Services (src/services/)¶

    Business logic lives here. Never write DB queries directly in components.

    Service Responsibility
    AcademicService.ts Grade calculation, class positions, report card generation
    FeeService.ts Ledger generation, payment allocation, fee schedule
    PayrollService.ts Salary calculations, WHT deductions, pension
    NotificationService.ts Email (Resend) and SMS (Termii) dispatch
    WalletService.ts Parent wallet fund/debit operations
    AuditService.ts Compliance event logging to audit_logs
    AttendanceService.ts Staff & student attendance CRUD
    StoreService.ts Inventory management, sales recording
    SupplierService.ts Supplier management, bill payment, WHT
    ExpenseService.ts School expense recording
    ReportCardService.ts PDF report card generation
    AnnouncementService.ts School-wide broadcast management
    CurriculumService.ts Lesson plan uploads, approval workflow
    QuizService.ts CBT quiz creation, student exam sessions
    AppraisalService.ts Staff performance review workflow

    Hooks (src/hooks/)¶

    Hook What it does
    useSchoolSettings Returns current school context (schoolId, session, term, subdomain)
    useAuth Returns current user session and role
    useOfflineSync Manages the offline attendance queue with auto-sync

    Key Utilities (src/utils/)¶

    File Purpose
    subdomain.ts slugifySubdomain() and buildSubdomainUrl()
    idCardGenerator.ts PDF ID card generation (student + staff)

    Backend: server/src/¶

    Entry Point¶

    server/src/index.ts — creates the Express app, registers middleware, and mounts all routes under /api.

    // Route mounting order matters for Express
app.use('/api', authRouter);        // Auth endpoints (no middleware)
app.use('/api', dbProxyRouter);     // DB proxy (uses authMiddleware internally)
app.use('/api', onboardSchoolRouter);
// ... etc

    Route Files¶

    File Mounted At Auth Required Description
    auth.ts /api/auth/* No (except /session) Signup, login, demo-login, session, accept-invite
    dbProxy.ts /api/db-proxy Yes Universal table CRUD proxy
    onboardSchool.ts /api/onboard-school Yes (admin) Creates school record + Monnify subaccount
    generateVirtualAccount.ts /api/generate-virtual-account Yes Creates Monnify reserved account for student
    monnifyWebhook.ts /api/monnify-webhook HMAC signature Processes incoming payment notification
    webhookPayment.ts /api/webhook-payment HMAC signature Alternate webhook route
    verifyPayment.ts /api/verify-payment Yes Server-to-server payment verification
    inviteUser.ts /api/invite-user Yes (admin) Staff invitation via JWT link
    promoteStudents.ts /api/promote-students Yes (admin) Batch class promotion
    sendEmail.ts /api/send-email Internal Resend API proxy
    sendSms.ts /api/send-sms Internal Termii API proxy

    Auth Helper (server/src/auth-helper.ts)¶

    Export Type Purpose
    hashPassword(pwd) async fn bcrypt hash (10 rounds)
    comparePassword(pwd, hash) async fn bcrypt compare
    generateToken(payload, expiry?) fn Signs JWT (default 7d)
    verifyToken(token) fn Decodes and verifies JWT
    authMiddleware Express middleware Validates Authorization: Bearer header
    AuthenticatedRequest TypeScript interface req.user: TokenPayload
    TokenPayload TypeScript interface { id, email, role, schoolId }

    Database: src/db/¶

    Schema (src/db/schema.ts)¶

    The single source of truth for all table definitions.

    Table Purpose Key Columns
    schools Primary tenant entity id, subdomain, monnify_subaccount_code, current_session, current_term
    profiles User accounts id, school_id, role, email, password_hash
    students Student directory id, school_id, student_id, class, virtual_account_number, outstanding_balance
    master_fee_schedule Fee rulebook per class school_id, class_level, fee_category_id, amount, is_mandatory
    student_ledgers Individual student debt student_id, fee_name, amount, amount_settled, is_paid
    transactions Immutable payment log student_id, amount, transaction_reference, platform_fee
    staff Staff directory school_id, role, nin, trcn_number, monthly_salary
    staff_attendance Staff clock-in records staff_id, date, clock_in_coordinates, photo_url
    student_attendance Student roll-call student_id, date, status, marked_by
    academic_records Grades student_id, subject_id, ca_score, exam_score, total_score
    notifications In-app + external alerts school_id, type, channel, is_read
    audit_logs Compliance event log school_id, user_id, action, entity_type
    payroll_runs Monthly payroll batches school_id, month, year, status
    parent_wallets Parent prepaid wallet student_id, balance

    Setup Scripts¶

    Script Command When to Run
    schema.ts npm run db:push After any schema change
    setup_rls.ts npm run db:rls After adding new tables
    setup_triggers.ts npm run db:triggers After schema changes affecting triggers
    seed.ts npm run db:seed Fresh dev environment setup
    promote_admin.ts npx tsx src/db/promote_admin.ts Elevating super-admin
    prod_init.ts npm run db:init First-time production initialization

    Incremental Migration Scripts¶

    These are one-time scripts used to add features to an existing database without a full schema reset.

    Script Adds
    migrate_wallets.ts parent_wallets, parent_wallet_transactions
    migrate_cbt.ts quizzes, quiz_questions, quiz_attempts
    migrate_comm.ts announcements, broadcasts
    migrate_curriculum.ts curriculum_documents
    migrate_subdomains.ts subdomain column to schools
    migrate_wht.ts WHT columns to supplier_bills

    Configuration Files¶

    File Purpose
    vite.config.ts Vite dev server (port 8080), SWC plugin, PWA, esbuild drop console in prod
    tailwind.config.ts Tailwind CSS theme with custom Remitpro design tokens
    drizzle.config.ts Drizzle ORM: schema path, migrations output dir, DB credentials
    tsconfig.json TypeScript root config (references app + node configs)
    tsconfig.app.json Frontend TypeScript settings (@/* path alias → ./src/*)
    components.json shadcn/ui config (style: default, base colour: slate)
    vercel.json SPA rewrites + security headers (HSTS, X-Frame-Options, CSP)
    Dockerfile Docker image for Express backend on Fly.io (Node 20 slim)
    fly.toml Fly.io app config (region: AMS, internal port: 5000, force HTTPS)
    .npmrc legacy-peer-deps=true to resolve peer conflict
    .env.example Template for all required environment variables

    Design System¶

    The frontend uses Tailwind CSS + shadcn/ui on a custom "Indigo-Night" design system.

    Token Value Usage
    primary hsl(243 75% 59%) Buttons, active states, links
    accent Indigo Highlights, badges
    Surface Glassmorphism (backdrop-blur-xl) Card backgrounds
    Dark mode next-themes (class strategy) System or manual toggle
    Font System sans-serif + custom CSS No Google Fonts in production bundle

    All shadcn/ui components live in src/components/ui/ and are auto-generated. Do not manually edit them — use npx shadcn add <component> to add new ones.
