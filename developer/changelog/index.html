    Changelog¶

    All notable changes to Remitpro are documented here.

    Format follows Keep a Changelog, versioning follows Semantic Versioning.

    [2.5.2] — 2026-06-24¶

    Fixed¶

    • Demo Login — Replaced fragile signup fallback with idempotent POST /api/auth/demo-login endpoint. Repeated demo logins no longer fail due to unique constraint violations on the greenwood school record. The endpoint atomically upserts the school and profile on every call.
    • db:push command — Upgraded drizzle-kit from v0.18.1 to v0.31.10 to restore the push subcommand that was removed in intermediate versions.

    Improved¶

    • MkDocs — Full enterprise documentation upgrade: navigation instant loading, toc.follow, social links, repo_url, cookie consent, custom CSS/JS, feedback widget, and new developer pages (Onboarding, Codebase Map, DB Schema, Troubleshooting, Contributing, Changelog).

    [2.5.1] — 2026-06-20¶

    Added¶

    • Staff Appraisal System — Formal termly performance review workflow with objective tracking and score history.
    • Curriculum Repository — Teachers upload lesson plans and schemes of work; admins approve and publish.
    • Bulk Grade Import — CSV/Excel template upload for CA and Exam scores directly in the Grades module.

    Fixed¶

    • Incorrect audit log labels in AttendanceService.ts causing misleading compliance records.
    • Student daily roll-call unique index constraint to prevent duplicate entries per day.

    [2.5.0] — 2026-06-15¶

    Added¶

    • Parent Wallet Engine — Internal prepaid wallet for every student. Parents fund via Monnify VA; admins can debit wallet for fees, store items, or transport.
    • WHT (Withholding Tax) Support — Supplier bills now include WHT rate, computed deduction, and net_amount as a PostgreSQL generated column.
    • Batch Payslip Download — ZIP generation for entire payroll runs so admins can download all payslips at once.

    Changed¶

    • current_session default updated from 2023/2024 to 2024/2025 in schema.

    [2.4.0] — 2026-06-05¶

    Added¶

    • Computer Based Testing (CBT) — Quiz builder for teachers with timed multi-choice questions. Student exam mode with auto-scoring.
    • Broadcast Center — Admin dashboard for SMS/Email blasts to filtered audiences (class, all parents, all staff).
    • Remittance Intelligence Hub — Super-admin dashboard showing ₦200/transaction earnings across all schools, projected revenue, and collection rate benchmarks.

    [2.3.0] — 2026-05-20¶

    Added¶

    • PDF ID Card Engine — Generates student and staff ID cards in PVC vertical format with school branding.
    • Global Fast-Search — ⌘K / Ctrl+K shortcut searches students, staff, and fees from any page.
    • Custom School Branding — Admins can upload school logos that appear on PDFs, portals, and ID cards.

    Improved¶

    • Dark mode transitions — smoother glassmorphism animations and high-contrast mode compliance.

    [2.2.0] — 2026-05-10¶

    Added¶

    • Smart Attendance (Biometric) — Geofencing + facial liveness check for staff clock-in via mobile browser. GPS coordinates and attendance photo stored and displayed in the admin dashboard.
    • Student Daily Roll-Call — Teachers mark daily student attendance from any device.
    • Offline-First Attendance — Service Worker queues roll-call data locally when offline; auto-syncs when reconnected.

    Fixed¶

    • Race condition in virtual account generation where duplicate Monnify API calls caused duplicate account numbers.

    [2.1.0] — 2026-04-28¶

    Added¶

    • Express.js Backend Migration — All 10 Supabase Edge Functions (Deno) ported to Node.js/Express routes. Backend now deployed on Fly.io with direct Drizzle ORM connections to Neon PostgreSQL.
    • Custom JWT Authentication — Replaced Supabase GoTrue with bcryptjs + jsonwebtoken for full multi-tenant token control. client.ts intercepts all supabase.auth.* calls.
    • DB Proxy Route — Universal POST /api/db-proxy proxies all table CRUD with automatic school_id scoping, replacing PostgREST.

    Security¶

    • JWT now embeds school_id in every token payload, preventing cross-tenant data access at the application layer.

    [2.0.0] — 2026-04-15 ⚠️ Breaking¶

    Breaking Changes¶

    • Authentication system replaced: Supabase GoTrue → Custom JWT. Existing user sessions are invalidated. All users must log in again.
    • supabase.functions.invoke() now routes to Express backend. Edge Functions are deprecated.

    Added¶

    • Multi-tenant Subdomain System — Each school gets a unique subdomain (school.remitpro.io). Tenant context is derived from URL hostname.
    • Payroll System — Full payroll run management with line items, status tracking, and PDF payslips.
    • Inventory & Store — School store with SKU management, stock logging, and sales recording.

    [1.5.0] — 2026-03-30¶

    Added¶

    • Academic Result Publishing — Admin "Publish Results" toggle gates parent portal access. Parents can download PDF report cards.
    • Class Positions — Automatic ranking calculation (1st, 2nd, 3rd...) after grade entry.
    • Behavioral Skills Ratings — Teacher assessments for soft skills (Punctuality, Neatness, Handwriting) on student report cards.

    [1.4.0] — 2026-03-15¶

    Added¶

    • Supplier Management — Track suppliers, issue bills, record partial payments.
    • Expense Tracking — Log operational expenses with categories, receipts, and approval status.
    • P&L Dashboard — Real-time Profit & Loss: Income (collected) − Expenses − Payroll.

    [1.3.0] — 2026-03-01¶

    Added¶

    • Staff Invite System — Role-based staff invitations via JWT-secured email links. Staff set their own password on first login.
    • RBAC Enforcement — Role-based access control enforced at the UI level and API middleware.
    • Audit Logs — All sensitive mutations (fee changes, manual payments, role promotions) logged to audit_logs table.

    [1.2.0] — 2026-02-15¶

    Added¶

    • Monnify Virtual Accounts — Per-student dedicated Wema Bank virtual accounts provisioned via Monnify API.
    • Webhook Payment Processing — Monnify webhook validates HMAC-SHA512 signature, updates ledger, settles ₦200 platform fee.
    • Payment Receipt Emails — Automated receipts via Resend API on payment confirmation.

    [1.1.0] — 2026-02-01¶

    Added¶

    • Master Fee Schedule — Define fees by class level and category. Mandatory vs optional flagging.
    • Student Ledgers — Automatic debt generation from fee schedule when term starts.
    • Bulk CSV Import — Upload student lists via CSV with validation and duplicate detection.

    [1.0.0] — 2026-01-15 🎉 Initial Release¶

    Added¶

    • Multi-tenant school management SaaS foundation
    • School onboarding wizard (2-step: profile + bank account)
    • Student directory with CRUD operations
    • Supabase Auth + PostgreSQL (Neon) with Row Level Security
    • React + Vite + shadcn/ui + Tailwind CSS frontend
    • PWA (Progressive Web App) with service worker
    • "Indigo-Night" design system with dark mode support
    • Super-Admin platform dashboard
    • Parent Portal (no-login access via Student ID + phone)

    For security vulnerability reports, email security@remitpro.io privately.
