    API Reference & Integration Guide 🔌¶

    Remitpro provides a suite of internal and external API endpoints via a Node.js/Express server to power the multi-tenant school operations, payment services, and messaging workflows.

    🏗️ Base Configuration¶

    • Local Development URL: http://localhost:5001/api
    • Production URL: https://[YOUR_EXPRESS_HOST]/api
    • Authentication: Custom JWT Auth Token (sent via Authorization: Bearer <JWT>). User login, registration, and sessions are completely database-driven and managed by the Express microservices.

    🛠️ API Endpoints¶

    1. Register School & Owner (POST /api/auth/signup)¶

    Registers a new school tenant and creates the default owner account in the profiles table. - Method: POST - Payload: 

    {
  "email": "owner@school.com",
  "password": "SecurePassword123!",
  "fullName": "Adebayo Adesina",
  "schoolName": "Lagos Academic Center",
  "subdomain": "lagos-academic",
  "phone": "+2348031112222",
  "address": "12 Broad St, Lagos"
}
    - Response: 200 OK 
    {
  "success": true,
  "token": "eyJhbGciOiJI...",
  "user": {
    "id": "uuid-string-here",
    "email": "owner@school.com",
    "fullName": "Adebayo Adesina",
    "role": "owner",
    "schoolId": "uuid-school-id"
  }
}

    2. User Login (POST /api/auth/login)¶

    Authenticates user credentials and issues a custom JWT access token. - Method: POST - Payload: 

    {
  "email": "user@school.com",
  "password": "SecurePassword123!"
}
    - Response: 200 OK (returns success, token, and user profile data).

    3. Get Active Session (GET /api/auth/session)¶

    Retrieves the logged-in user profile from the database based on the provided JWT bearer token. - Method: GET - Headers: Authorization: Bearer <JWT> - Response: 200 OK (returns the user profile data).

    4. Accept User Invitation (POST /api/auth/accept-invite)¶

    Completes registration for a new user by verifying their invitation JWT token and saving their password credentials. - Method: POST - Payload: 

    {
  "token": "invitation-jwt-token-here",
  "password": "UserChosenPassword123!",
  "fullName": "Chioma Nze"
}
    - Response: 200 OK (returns session token and user profile data).

    5. Demo Login (POST /api/auth/demo-login)¶

    Provisions (or re-activates) the built-in Greenwood International School demo workspace and returns a valid session token. This endpoint is idempotent — it can be called repeatedly without errors regardless of DB state (upserts school + profile, always refreshes password hash). - Method: POST - Payload: (none required) - Response: 200 OK 

    {
  "success": true,
  "token": "eyJhbGciOiJI...",
  "user": {
    "id": "uuid-string-here",
    "email": "info@greenwood.edu.ng",
    "fullName": "Demo Admin",
    "role": "owner",
    "schoolId": "uuid-school-id",
    "schoolSubdomain": "greenwood"
  }
}

    Demo Credentials

    Email: info@greenwood.edu.ng | Password: password123 The demo workspace is pre-seeded with students, fees, grades, and expenses.

    6. Onboard School (POST /api/onboard-school)¶

    Initializes a new school workspace, provisions their tenant record in Neon, and sets up a Monnify bank sub-account for split remittances. - Method: POST - Headers: Authorization: Bearer <Admin_JWT> - Payload: 

    {
  "name": "Greenwood International School",
  "email": "admin@greenwood.edu.ng",
  "phone": "+2348031234567",
  "address": "12 High School Rd, Lagos",
  "bankName": "Wema Bank",
  "accountNumber": "0123456789",
  "accountName": "Greenwood Edu Services"
}
    - Response: 200 OK 
    {
  "success": true,
  "schoolId": "uuid-string-here",
  "subAccountCode": "MFY_SUB_..."
}

    7. Generate Virtual Account (POST /api/generate-virtual-account)¶

    Provisions a dedicated bank virtual account (Wema/Rolez) for a student via the Monnify API. - Method: POST - Headers: Authorization: Bearer <User_JWT> - Payload: 

    {
  "studentId": "uuid-student-id",
  "schoolId": "uuid-school-id"
}
    - Response: 200 OK returning details of the reserved bank account.

    8. Invite Staff User (POST /api/invite-user)¶

    Initiates the onboarding/RBAC invite flow for bursars and other staff members by issuing a secure JWT-based invitation token. - Method: POST - Headers: Authorization: Bearer <Admin_JWT> - Payload: 

    {
  "email": "bursar@greenwood.edu.ng",
  "role": "bursar",
  "schoolId": "uuid-school-id"
}
    - Response: 200 OK (invitation link generated and sent to the staff member).

    9. Verify Monnify Payment (POST /api/verify-payment)¶

    Performs a direct server-to-server transaction verification against the Monnify API and processes the student ledger sync. - Method: POST - Headers: Authorization: Bearer <User_JWT> - Payload: 

    {
  "transactionReference": "SP-TX-1004321"
}
    - Response: 200 OK with verified payment confirmation details.

    10. Monnify Webhook Handler (POST /api/monnify-webhook)¶

    Processes transaction notifications securely received from Monnify's transaction servers. - Method: POST - Headers: monnify-signature (HMAC-SHA512 checksum validating the payload against the merchant secret) - Payload: Standard Monnify webhook JSON notification payload containing transactionReference, amountPaid, paymentStatus, etc. - Response: 200 OK (Processes splits, sets student ledger status to Paid, and emits system alerts).

    11. Promote Students (POST /api/promote-students)¶

    Performs batch promotion of students to the next class levels and rolls over the term cycle. - Method: POST - Headers: Authorization: Bearer <Admin_JWT> - Payload: 

    {
  "schoolId": "uuid-school-id",
  "promotions": [
    { "fromClass": "JSS1", "toClass": "JSS2" },
    { "fromClass": "JSS2", "toClass": "JSS3" }
  ]
}
    - Response: 200 OK (Returns summary of promoted student counts).

    12. Send Proxy Email (POST /api/send-email)¶

    Enables transactional mailing (via Resend) without exposing server API keys to frontend assets. - Method: POST - Payload: 

    {
  "to": "recipient@domain.com",
  "subject": "Greenwood Onboarding Invite",
  "html": "<p>Invite body...</p>"
}
    - Response: 200 OK

    13. Send Proxy SMS (POST /api/send-sms)¶

    Enables transactional mobile SMS sending (via Termii) to parents and staff. - Method: POST - Payload: 

    {
  "to": "+2348000000000",
  "message": "Dear Parent, fee payment of ₦50,000 was successfully processed."
}
    - Response: 200 OK

    📡 Webhook/Realtime Specifications¶

    If you wish to subscribe to payment events asynchronously, configure listener endpoints against Supabase Realtime listening on the transactions table.

    Event Schema: PAYMENT_SUCCESS¶

    Emitted automatically when a virtual account payment settles. 

    {
  "event": "PAYMENT_SUCCESS",
  "data": {
    "student_id": "uuid-student-id",
    "amount": 48500.00,
    "transaction_ref": "MNFY-98217340",
    "paid_at": "2026-06-20T12:00:00Z"
  }
}

    🔐 Rate Limiting & Quotas¶

    • Standard Tier: 120 requests / minute per tenant.
    • Enterprise Tier: Scalable clusters with custom throughput settings.
