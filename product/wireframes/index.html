    Remitpro Enterprise UI/UX Wireframes¶

    Document Standard

    This document follows enterprise wireframe conventions: every screen uses production-realistic data, annotated component interactions, explicit database field bindings, error states, and an onboarding flow. All three user personas are covered: School Admin, Parent, and Super-Admin (Platform Owner).

    🔍 Annotations — School Admin Finance Console¶

    Component Behaviour DB Binding
    KPI Cards Computed live from transactions table on page load SUM(amount) WHERE school_id + term_id + status='success'
    Weekly Bar Chart Groups transactions.created_at by ISO week. Gold bars = weeks < 60% expected transactions.created_at, fee_items.amount * student_count

    SMS Template Uses [Student Name] and [Amount] tokens replaced by notification service students.first_name, computed balance_due

    🔍 Annotations — Parent Portal¶

    🎨 5. Enterprise Design System Reference¶

    5.1 Colour Tokens¶

    Token HSL Value Usage
    --rp-purple hsl(263, 70%, 50%) Primary actions, active states, branding
    --rp-indigo hsl(243, 75%, 59%) Gradient pairs, chart fills
    --rp-gold hsl(43, 96%, 56%) Warning states, below-target indicators
    --rp-surface hsl(240, 10%, 98%) Background surfaces, sidebar bases
    #27c93f — Success, paid, operational
    #ff5f56 — Error, blocked, overdue
    #ffbd2e — Moderate warning

    5.2 Typography Scale¶

    Level Size Weight Usage
    Page Title 1rem 800 Section headings
    Card Stat 1.5rem 800 KPI metric values
    Card Title 0.68rem 700 Card labels (uppercase)
    Body 0.73–0.8rem 400–500 Table rows, descriptions
    Hint / Caption 0.6–0.65rem 600 Secondary metadata
    Monospace JetBrains Mono 700 Account numbers, codes

    5.3 Interaction States¶

    State CSS Class Visual
    Default — 1px border, rgba(0,0,0,0.04) shadow
    Hover (card row) :hover hsla(263,70%,50%,0.03) background tint
    Active (sidebar) .active Purple left-border 3px, purple text, light bg
    Focus (input) :focus var(--rp-purple) border + 3px ring glow
    Hover (btn primary) :hover Darker purple + elevated shadow
    Hover (chart bar) :hover opacity: 0.78, cursor: crosshair

    5.4 Accessibility¶

    WCAG 2.1 Requirements

    • All badge text must maintain 4.5:1 contrast ratio vs background.
    • Chart bars must have aria-label attributes with the value in production (e.g., aria-label="Week 4: ₦4.5M collected").
    • The virtual account Copy button must announce "Account number copied" to screen readers via aria-live="polite".
    • All interactive elements must be keyboard-focusable in tab order.
