    Database Schema Reference¶

    Complete reference for all 25+ tables in the Remitpro Neon PostgreSQL database.

    Schema Source

    The authoritative schema is defined in src/db/schema.ts using Drizzle ORM. This document is derived from that file.

    Core Tenant Tables¶

    schools — Primary tenant entity¶

    Column Type Notes
    id uuid PK Auto-generated
    name text NOT NULL School display name
    subdomain text UNIQUE URL slug (e.g. greenwood)
    address text Physical address
    phone text Contact number
    email text Admin contact email
    logo_url text URL to school logo in Supabase Storage
    monnify_contract_code text Monnify contract reference
    monnify_public_key text Used by Monnify Web SDK
    bank_name text Settlement bank
    account_number text Settlement NUBAN
    account_name text Account name (must match BVN)
    monnify_subaccount_code text School's Monnify sub-account
    subscription_tier text free | growth | enterprise
    subscription_status text active | past_due | canceled
    max_students numeric Default: 50
    current_session text NOT NULL e.g. 2024/2025
    current_term text NOT NULL TERM_1 | TERM_2 | TERM_3
    are_results_published boolean Default: false
    geofence_lat numeric(10,7) Staff clock-in geofence
    geofence_lng numeric(10,7) Staff clock-in geofence
    geofence_radius numeric Meters. Default: 100
    created_at timestamptz Auto
    updated_at timestamptz Auto

    profiles — User accounts¶

    Column Type Notes
    id uuid PK
    school_id uuid FK → schools NULL for super-admin
    role text owner | admin | bursar | teacher | super_admin
    full_name text
    email text Used for login
    password_hash text bcrypt hash (10 rounds)
    created_at timestamptz
    updated_at timestamptz

    Role Meanings

    • owner — Created during onboarding; has full school access
    • admin — Equivalent to owner for operational purposes
    • bursar — Finance-only access
    • teacher — Academics-only access
    • super_admin — Platform-wide access; bypasses RLS

    Finance Tables¶

    fee_categories — Fee type definitions¶

    Column Type Notes
    id uuid PK
    school_id uuid FK → schools CASCADE
    name text e.g. Tuition, Bus Service, Uniform
    description text
    created_at timestamptz

    master_fee_schedule — Fee rulebook¶

    Defines what each class level should pay for a given fee category.

    Column Type Notes
    id uuid PK
    school_id uuid FK → schools CASCADE
    class_level text e.g. JSS1, SS2, Primary 3
    fee_category_id uuid FK → fee_categories
    amount numeric(12,2) Fee amount in Naira
    is_mandatory boolean Default: true
    frequency enum once_per_term | once_per_session | one_time_only
    created_at / updated_at timestamptz

    students — Student directory¶

    Column Type Notes
    id uuid PK
    school_id uuid FK → schools CASCADE Multi-tenant key
    first_name text
    last_name text
    student_id text School-assigned ID (e.g. STU/102)
    class text Current class level
    email text Optional
    parent_phone text Used for portal login + SMS
    parent_name text
    total_fees numeric(12,2) Sum of all billed fees
    outstanding_balance numeric(12,2) Current unpaid amount
    virtual_account_number text UNIQUE Monnify account number
    virtual_account_bank text e.g. Wema Bank
    virtual_account_name text Account name on Monnify
    account_reference text UNIQUE Monnify account reference
    payment_status enum paid | partial | owed

    Indexes: (school_id, student_id) unique index, created_at index.

    student_ledgers — Individual debt records¶

    One row per fee item per student per term.

    Column Type Notes
    id uuid PK
    school_id uuid FK
    student_id uuid FK → students CASCADE
    fee_schedule_id uuid FK → master_fee_schedule May be NULL for manual entries
    fee_name text Snapshot of fee name at time of billing
    amount numeric(12,2) Original billed amount
    amount_settled numeric(12,2) Amount paid so far
    is_paid boolean True when amount_settled >= amount
    term text e.g. TERM_1

    Indexes: (school_id), (student_id), (school_id, is_paid), created_at.

    transactions — Immutable payment log¶

    Never update or delete rows in this table.

    Column Type Notes
    id uuid PK
    student_id uuid FK → students CASCADE
    school_id uuid FK → schools CASCADE
    amount numeric(12,2) Gross payment received
    transaction_reference text UNIQUE Monnify transaction ref
    payment_reference text Bank payment reference
    paid_on timestamptz When payment was made
    payment_method text e.g. ACCOUNT_TRANSFER
    bank_name text Payer's bank
    customer_name text Payer name
    balance_before numeric(12,2) Student balance before payment
    balance_after numeric(12,2) Student balance after payment
    monnify_transaction_hash text Monnify HMAC hash
    platform_fee numeric(12,2) ₦200 platform deduction
    status text Default: successful

    Academic Tables¶

    subjects¶

    Column Type Notes
    id uuid PK
    school_id uuid FK
    name text e.g. Mathematics
    code text e.g. MTH
    coefficient numeric Grade weighting. Default: 1
    category text e.g. Sciences, Languages
    is_active boolean

    academic_records — Student grades¶

    Column Type Notes
    id uuid PK
    school_id uuid FK
    student_id uuid FK → students CASCADE
    subject_id uuid FK → subjects CASCADE
    session text e.g. 2024/2025
    term text e.g. TERM_1
    ca_score numeric Continuous assessment (0–40)
    exam_score numeric Exam score (0–60)
    total_score numeric CA + Exam
    grade text Computed: A, B, C, D, F
    remark text e.g. Excellent
    comment text Teacher comment

    Unique index: (student_id, subject_id, session, term)

    behavioral_skills & behavioral_records¶

    Soft skill assessments (Punctuality, Neatness, Politeness, Handwriting).

    Column Notes
    behavioral_skills.name Skill name
    behavioral_skills.max_score Maximum rating (default: 5)
    behavioral_records.rating Teacher rating
    behavioral_records.comment Optional teacher note

    Unique index on behavioral_records: (student_id, skill_id, session, term)

    class_positions — Term rankings¶

    Column Notes
    student_id
    session / term Academic period
    total_score Sum of all subjects
    average_score Mean score
    position Rank in class (1 = first)
    out_of Total students in class

    HR & Payroll Tables¶

    staff¶

    Column Notes
    role teacher | bursar | admin | support
    designation Job title
    employment_type full_time | part_time | contract
    status active | resigned | terminated
    nin National Identity Number
    tin Tax Identification Number
    pfa_name Pension Fund Administrator
    rsa_number Retirement Savings Account
    trcn_number Teachers Registration Council of Nigeria
    monthly_salary numeric(12,2)
    bank_name / account_number Payroll settlement bank

    staff_attendance¶

    Column Notes
    date text format YYYY-MM-DD
    check_in Timestamp with timezone
    check_out Timestamp with timezone
    status present | absent | late
    clock_in_coordinates JSON {"lat":...,"lng":...}
    photo_url Supabase Storage URL
    verification_method biometric | manual

    Unique index: (staff_id, date)

    payroll_runs & payroll_items¶

    Column Notes
    payroll_runs.month / .year Payroll period
    payroll_runs.status draft | paid
    payroll_items.staff_id Individual payslip
    payroll_items.amount Gross salary for this run
    payroll_items.status pending | paid

    Inventory & Supplier Tables¶

    store_categories, store_items, store_stock_logs¶

    Table Key Columns
    store_items name, price, stock_quantity, reorder_level, sku
    store_stock_logs item_id, change_quantity, reason, performed_by

    suppliers & supplier_bills¶

    Column Notes
    supplier_bills.wht_rate Withholding Tax rate (e.g. 5.00%)
    supplier_bills.wht_amount Computed WHT deduction
    supplier_bills.net_amount Generated column: amount_total - wht_amount
    supplier_bills.status open | partial | paid

    System Tables¶

    notifications¶

    Column Notes
    type system_alert | payment | academic
    channel in_app | email | sms
    recipient admin (for dashboard bell) or email/phone for external
    is_read Boolean — cleared when admin reads
    status pending | sent | failed
    error_message Delivery error from Resend/Termii

    audit_logs¶

    Column Notes
    action e.g. student_created, fee_modified, payment_manual_entry
    entity_type e.g. student, fee, payroll
    entity_id UUID of the affected record
    details JSON string with before/after or context

    Enums¶

    Enum Values
    payment_status paid, partial, owed
    fee_frequency once_per_term, once_per_session, one_time_only

    Indexing Strategy¶

    Performance-critical indexes defined in the schema:

    Index Table Columns Purpose
    school_id_student_id_idx students (school_id, student_id) UNIQUE Fast student lookup by school
    ledger_unpaid_idx student_ledgers (school_id, is_paid) Debtors list query
    transactions_school_idx transactions (school_id) School revenue reports
    audit_school_idx audit_logs (school_id) Audit trail per school
    staff_daily_idx staff_attendance (staff_id, date) UNIQUE Prevent duplicate clock-ins
    student_daily_idx student_attendance (student_id, date) UNIQUE Prevent duplicate roll-call
    student_position_idx class_positions (student_id, session, term) UNIQUE One position per period
